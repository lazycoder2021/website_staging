Various Microsoft 365 Administrator Roles

Microsoft 365 is a powerful suite of cloud-based productivity tools that empowers organizations to collaborate, communicate, and manage their data securely. To ensure smooth operation and security, Microsoft provides a range of administrative roles that allow businesses to delegate various responsibilities to different team members. These roles help in maintaining the integrity and security of the organization's Microsoft 365 environment. In this article, we'll explore some of the key Microsoft 365 admin roles.


Global Administrator

The Global Administrator is the highest-level administrative role in Microsoft 365. They have full access to all administrative features and settings, making them responsible for managing user accounts, domains, licenses, and security settings. Global Administrators can delegate some tasks to other admin roles as needed.


User Administrator

User Administrators focus on managing user accounts, including creating, modifying, and deleting user profiles. They also reset user passwords, manage group memberships, and assign licenses to users. This role is ideal for HR professionals or department managers who handle employee onboarding and offboarding.


Exchange Administrator

As the name suggests, Exchange Administrators are responsible for managing email-related tasks. They handle settings for Exchange Online, manage mailboxes, distribution groups, and public folders, and troubleshoot email issues.


Sharepoint Administrator

SharePoint Administrators are tasked with managing SharePoint Online, including document libraries, permissions, and site collections. They ensure the efficient organization and sharing of documents and collaborate with teams using SharePoint.


Teams Administrator

Teams Administrators focus on Microsoft Teams, the collaboration and communication platform. They manage user access to Teams, configure meeting and chat settings, and oversee security and compliance within the platform.


Security Administrator

Security Administrators are responsible for safeguarding Microsoft 365 against cyber threats. They manage security policies, implement multi-factor authentication, and monitor security alerts. This role is vital in protecting sensitive data and ensuring regulatory compliance.


Compliance Administrator

Compliance Administrators handle data governance and compliance within Microsoft 365. They configure data retention policies, manage eDiscovery, and ensure that the organization adheres to legal and regulatory requirements.


Billing Administrator

Billing Administrators manage the financial aspects of the Microsoft 365 subscription, including billing, purchasing, and license management. They help optimize licensing costs and track expenses.


Intune Administrator

Intune Administrators focus on managing mobile devices and apps within the organization. They configure device settings, enforce security policies, and oversee mobile application management.


Helpdesk Administrator

Help Desk Administrators have limited administrative access to provide support and troubleshoot issues for end-users. They can reset passwords, manage user accounts, and assist with basic tasks, reducing the workload on other administrators.


How to Assign Administrator Role Using Microsoft 365 Admin Center?


  1. Log into Microsoft 365 using this link: https://login.microsoftonline.com/
  2. Go to the Active Users page within Microsoft 365 Admin Center by clicking this link: https://admin.microsoft.com/#/users
  3. Select the user you wish to assign the administrator role to by clicking the display name.
  4. Click Manage Roles option under the Roles section. This opens the Manage admin roles pane.
  5. Select Admin Center access option and then choose the admin role you wish to assign and Save the changes.

How to Assign Administrator Role Using Microsoft Entra Admin Center?


  1. Log into Microsoft 365 using this link: https://login.microsoftonline.com/
  2. Go to the Users page within the Microsoft Entra Center by clicking this link: https://entra.microsoft.com/#view/Microsoft_AAD_UsersAndTenants/UserManagementMenuBlade/~/AllUsers/menuId/
  3. Select the user you wish to assign the administrator role to by clicking the display name.
  4. The user page opens up as shown in the image. Select Assigned Roles under Manage.
  5. Click the Add Assignments button.
  6. The Directory Roles pane opens up. Select admin role you wish to assign and then click Add.

Final Thoughts

By assigning these various Microsoft 365 admin roles, organizations can distribute responsibilities effectively and maintain the security and functionality of their cloud-based environment. It's important to carefully consider which roles are necessary for your organization and assign them accordingly to ensure a well-managed and secure Microsoft 365 experience.
